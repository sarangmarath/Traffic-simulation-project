About Project

The aim of this project is to design and simulate a traffic light control system that optimizes the timing of signals to reduce congestion and improve traffic flow. By applying concepts of CPU scheduling and pipelining, the project models how vehicles are managed at intersections, similar to how processes are scheduled and executed in a computer system.

Aim

  • To design an efficient traffic light controller.
  • To optimize signal timing using CPU scheduling concepts.
  • To reduce congestion and waiting time at intersections.
  • To simulate traffic flow using code and circuit design.

Objectives

  • Apply Contro Unit based concept to traffic signals.
  • Use pipelining for parallel light control.
  • Minimize waiting time and avoid deadlocks (traffic jams).
  • Implement both circuit-based and code-based models.

CPU Scheduling Analogy

In our model, vehicles are treated as processes, while the traffic light represents the CPU. When the light turns green, it is equivalent to a process being executed. A red light means the process is waiting. Different scheduling algorithms can be used to decide which lane gets the green signal, ensuring fairness or priority-based execution.
